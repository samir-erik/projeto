ESCOPO DO PROJETO – SITE DE NOTÍCIAS
1. Objetivo do Projeto Desenvolver um site de notícias com a finalidade de exibir informações
atualizadas em tempo real, permitindo que os usuários busquem e filtrem
conteúdos de forma simples e rápida.
2. Descrição Geral
O projeto consiste no desenvolvimento de um sistema web que apresenta notícias
de diversas categorias, utilizando uma API externa para obtenção dos dados.
As notícias exibidas no site serão fornecidas pela API GNews, garantindo
atualização constante das informações.
3. Funcionalidades (Requisitos Funcionais)
O sistema deverá permitir:
3.1 Exibir lista de notícias na página inicial
3.2 Buscar notícias por palavra-chave
 3.3 Filtrar notícias por categoria
3.4 Exibir detalhes das notícias (título, descrição, imagem e link)
3.5 Exibir imagens relacionadas às notícias
3.6 Navegação entre diferentes notícias (paginação simples)
4. Requisitos Não Funcionais
O sistema deverá:
4.1 Ser responsivo, adaptando-se a diferentes dispositivos (desktop e mobile)
4.2 Possuir tempo de resposta de até 2 segundos
4.3 Garantir segurança básica, como proteção da chave da API
4.4 Estar disponível para acesso online via navegador
4.5 Apresentar interface simples e de fácil utilização
5. Tecnologias Utilizadas
5.1 Front-end: HTML, CSS e JavaScript
5.2 Back-end: Python (Flask)
5.3 Banco de dados: Não utilizado (dados consumidos diretamente da API)
5.4 API externa: GNews
5.5 Hospedagem: Render
6. Estrutura do Sistema
O funcionamento do sistema ocorrerá da seguinte forma:
O  usuário acessa o site através do navegador
O front-end envia requisições ao back-end
O back-end realiza requisições à API GNews
A API retorna os dados em formato JSON
O back-end processa os dados e envia ao front-end
O front-end exibe as notícias ao usuário
7. Público-Alvo
O sistema é destinado a:
7.1 Usuários interessados em acompanhar notícias atualizadas
7.2 Pessoas que buscam acesso rápido a informações de diferentes categorias
8. Limitações do Projeto
O projeto não incluirá:
8.1 Sistema de cadastro ou login de usuários
 Criação própria de notícias
8.2 Armazenamento permanente de dados em banco
8.3 Funcionalidades avançadas como comentários ou interação social
9. Entregáveis
Ao final do projeto será entregue:
9.1 Código-fonte do sistema
9.2 Site funcional hospedado online
9.3 Documentação básica do projeto
9.4 Apresentação do sistema
